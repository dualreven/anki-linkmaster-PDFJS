# 执行任务的通用规则

1. 必须总是牢记你的执行流程, 任何时候都不能忘记!
2. 测试先行, 任何想法都必须经过代码验证才能落实,不知道不确定的事情必须验证!
3. 每次修改前, 必须先commit, 然后创建分支, 保存快照, 然后再进行修改
4. 如果修改导致更恶劣的情况发生(比如错误增多),必须立即退回修改
5. 修改通过后, 必须将当前分支合并到主分支, 并删除当前分支.
