**规范名称**: 测试规范优先原则
**规范描述**: 要求AI在进行测试前必须先查询相关测试规范，确保测试行为规范化、可追溯，避免任意测试。
**当前版本**: 1.0
**所属范畴**: 测试规范
**适用范围**: 所有AI进行的测试活动

**详细内容**:
- 测试前必须搜索 `docs/SPEC/TEST/` 目录下相关规范
- 如有规范，必须严格按规范执行测试
- 如无规范，必须询问用户测试方式
- 获得用户明确确认后方可进行测试

**正向例子**:
```bash
# AI行为示例：先查询规范
find docs/SPEC/TEST -name "*.md" -exec grep -l "PDF功能测试" {} \;
# 找到规范后按规范执行测试
```

**反向例子**:
```bash
# AI行为示例：错误做法，直接开始测试而不查询规范
# 直接修改代码进行测试，没有先检查是否有相关测试规范
